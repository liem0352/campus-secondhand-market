一、一键启动（推荐）
----------------------------------------------------------------

:: 1. 首次安装依赖（Python + 两个前端）
scripts\install-deps.bat

:: 2. 一键启动后端 + 卖家工作台 + 平台管理后台

:: 3. 完成后停止全部服务

二、分步启动（首次必须建库 + 迁移 + 种子数据）
----------------------------------------------------------------

:: 第 1 步：建库
"C:\Program Files\MySQL\MySQL Server 9.4\bin\mysql.exe" -u root -ptyb1124 < backend\scripts\create_mysql_db.sql
:: 库名：campus_market （字符集 utf8mb4）

:: 第 2 步：进入后端目录
cd backend

:: 第 3 步：生成并执行 migrations
C:\Users\liem\AppData\Local\Programs\Python\Python313\python.exe manage.py makemigrations market
C:\Users\liem\AppData\Local\Programs\Python\Python313\python.exe manage.py migrate

:: 第 4 步：初始化种子数据（4 个默认账号 + 5 个分类 + 20+ 商品）
C:\Users\liem\AppData\Local\Programs\Python\Python313\python.exe scripts\init_data_market.py

:: 第 5 步：启动后端（端口 8000）
cd ..
scripts\start-backend.bat

:: 第 6 步：启动卖家工作台（端口 3000，新终端）
scripts\start-frontend-web.bat

:: 第 7 步：启动平台管理后台（端口 5173，新终端）
scripts\start-frontend-admin.bat

:: 第 8 步：导入微信小程序
:: 微信开发者工具 -> 导入项目 -> 选择 miniprogram\
:: 修改 app.js 的 apiBase 指向 http://127.0.0.1:8000 或电脑局域网 IP
:: 勾选「不校验合法域名」

十、常见问题
----------------------------------------------------------------

1. Access denied for user 'root'
   -> 检查 backend\.env 中 DB_USER / DB_PASSWORD

2. Unknown database 'campus_market'
   -> 先执行 backend\scripts\create_mysql_db.sql 建库

3. No module named 'pymysql' / 'rest_framework'
   -> pip install -r backend\requirements.txt

4. 前端 3000 / 5173 端口连不上
   -> 检查 frontend-*\vite.config.{js,ts} 端口配置
   -> 端口被占用：netstat -ano | findstr ":3000"

5. 微信小程序请求失败
   -> 微信开发者工具 -> 详情 -> 勾选「不校验合法域名」
   -> app.js 的 apiBase 指向电脑局域网 IP（真机调试）

6. CORS 跨域报错
   -> 后端已默认开启 corsheaders，无需额外配置

7. AI 一键发布返回 mock
   -> 检查 LLM_API_KEY 是否配置
   -> 临时演示可设 LLM_MOCK_MODE=True

8. RabbitMQ 连接失败
   -> 确认 D:\RJ\RabbitMQ 已启动
   -> 检查 RABBITMQ_URL 和 5672 端口

9. 媒体文件 404
   -> 开发环境 runserver 自动处理 /media/
   -> 生产环境需 Nginx 配 location /media/ 指向 MEDIA_ROOT
