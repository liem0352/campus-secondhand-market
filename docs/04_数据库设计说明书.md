## 1. 设计概述

### 1.1 业务定位

校园二手交易平台是面向高校学生的二手物品交易 / 闲置流通 SaaS。核心实体为 **User（用户）**、**Product（商品）**、**Order（订单）**，并以 **Conversation/Message（私聊）**、**Favorite（收藏）**、**Review（评价）**、**Report（举报）**、**AuditLog（审计）**、**SystemSetting（系统配置）** 为支撑。

### 1.2 设计目标

- **可教学评审**：所有表/字段有中文 `verbose_name` 与 `help_text`，方便答辩讲解。
- **可生产扩展**：第三范式、显式外键、显式索引，DDL 与 ORM 一一对应。
- **可运营审计**：审核 / 调分 / 封禁等关键操作落 `market_audit_log` 不可变日志。
- **可观测**：复合索引覆盖首页瀑布流、类目页、我的发布、订单列表等核心读路径。

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| 第三范式 | 不引入汇总表 / 物化视图，统计全部走 SQL 聚合（v1 不引入） |
| 引用完整性 | 外键显式声明，删除策略按业务选择（CASCADE / SET_NULL / PROTECT） |
| 可审计 | 关键运营动作（审核 / 调分 / 封禁 / 处理举报）写 `market_audit_log` |
| 可配置 | 运行时参数入 `market_system_setting`（AI 提示词、限流阈值等） |
| 可索引 | 按查询路径（首页瀑布流、类目页、我的发布、订单列表）建组合索引 |
| 可注释 | 表/字段 COMMENT 完整，便于答辩与 DBA 交接 |

### 1.4 选型理由

| 维度 | 选择 | 理由 |
|------|------|------|
| 数据库 | MySQL 9.4 | 学校机房统一提供；事务稳定；utf8mb4 支持 emoji 表情 nickname（虽然本系统禁 emoji 图标） |
| 字符集 | utf8mb4 / utf8mb4_unicode_ci | 多语言校园场景；不区分大小写利于搜索 |
| 存储引擎 | InnoDB | 事务、外键、行锁必备 |
| ORM | Django 4.2 ORM | 与代码同源；migration 工具完善 |
| 缓存 | 本进程 LRU + 后期 Redis | v1 仅 LRU；v2 引入 Redis 缓存首页 Feed |

## 2. 命名与约定

### 2.1 标识符规范

| 项 | 规范 | 示例 |
|----|------|------|
| 表名 | 小写复数 snake_case + `market_` 前缀 | `market_product` |
| 字段名 | 小写 snake_case | `credit_score` |
| 主键 | `id` BIGINT AUTO_INCREMENT | `id` |
| 外键 | `<entity>_id` | `seller_id` |
| 时间 | `created_at` / `updated_at` | `created_at` |
| 布尔 | TINYINT(1)，0/1 | `is_certified` |
| 金额 | DECIMAL(10,2)，应用层校验 > 0 | `price` |
| 状态机 | VARCHAR(16) 枚举值 | `status='on_sale'` |
| 备注 | VARCHAR(128) | `audit_remark` |
| 文本 | TEXT（限制 max_length） | `description` |

### 2.2 状态机取值

| 实体 | 取值 | 说明 |
|------|------|------|
| Product.status | draft / pending / on_sale / pending_sold / sold / off_shelf | 详见 [CM-SRS-001 §4.4](#) |
| Order.status | requested / confirmed / shipping / completed / cancelled | 详见 §6 |
| Report.status | pending / warned / removed / banned / rejected | 详见 §6.3 |
| User.role | user / admin | 鉴权时使用 |

### 2.3 软删除策略

v1 **不引入** 软删除（`is_deleted`）。商品/订单靠状态机表达生命周期。理由：校园场景数据量小（千级用户、万级商品），硬删除 + 审计日志可回溯。

## 4. 表结构详细设计

### 4.1 `market_user` 用户

| 字段 | 类型 | 允许空 | 默认 | 索引 | 注释 |
|------|------|--------|------|------|------|
| id | BIGINT UNSIGNED | NO | AUTO | PK | 主键 |
| username | VARCHAR(32) | NO | - | UK | 登录名 |
| email | VARCHAR(254) | YES | '' | - | 邮箱（找回密码用，v1 不启用） |
| password | VARCHAR(128) | NO | - | - | Django pbkdf2 哈希 |
| first_name | VARCHAR(150) | YES | '' | - | AbstractUser 保留字段 |
| last_name | VARCHAR(150) | YES | '' | - | AbstractUser 保留字段 |
| school | VARCHAR(64) | YES | '' | - | 学校名（用于按学校过滤商品） |
| student_id | VARCHAR(32) | YES | '' | - | 学号（校园身份认证依据） |
| credit_score | INT | NO | 80 | - | 信用分（0-100，初始 80） |
| avatar | VARCHAR(512) | YES | '' | - | 头像 URL |
| bio | VARCHAR(128) | YES | '' | - | 个人简介 |
| role | VARCHAR(16) | NO | 'user' | IDX(role, is_active) | user / admin |
| is_certified | TINYINT(1) | NO | 0 | - | 是否完成校园身份认证 |
| is_active | TINYINT(1) | NO | 1 | - | 是否启用（封禁=False） |
| is_staff | TINYINT(1) | NO | 0 | - | Django Admin 登录 |
| is_superuser | TINYINT(1) | NO | 0 | - | 超管 |
| last_login | DATETIME | YES | NULL | - | 上次登录 |
| date_joined | DATETIME | NO | - | - | 注册时间 |
| created_at | DATETIME | NO | now() | - | 创建时间 |
| updated_at | DATETIME | NO | now() ON UPDATE | - | 更新时间 |

**唯一约束**：`username` UK；`email` 非唯一。

**索引**：`idx_mkt_user_role_active (role, is_active)`。

**典型查询**：

```sql
-- 首页按学校过滤
SELECT id, username, school, avatar, credit_score
FROM   market_user
WHERE  school = ? AND is_active = 1
LIMIT  20;
```

**DDL 关键片段**：

```sql
CREATE TABLE `market_user` (
    `id`           BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    `username`     VARCHAR(32) NOT NULL,
    `email`        VARCHAR(254) NOT NULL DEFAULT '',
    `password`     VARCHAR(128) NOT NULL,
    `school`       VARCHAR(64)  NOT NULL DEFAULT '',
    `student_id`   VARCHAR(32)  NOT NULL DEFAULT '',
    `credit_score` INT          NOT NULL DEFAULT 80,
    `avatar`       VARCHAR(512) NOT NULL DEFAULT '',
    `bio`          VARCHAR(128) NOT NULL DEFAULT '',
    `role`         VARCHAR(16)  NOT NULL DEFAULT 'user',
    `is_certified` TINYINT(1)   NOT NULL DEFAULT 0,
    `is_active`    TINYINT(1)   NOT NULL DEFAULT 1,
    `created_at`   DATETIME     NOT NULL,
    `updated_at`   DATETIME     NOT NULL,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_mkt_user_username` (`username`),
    KEY `idx_mkt_user_role_active` (`role`, `is_active`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='校园二手交易平台用户';
```

### 4.3 `market_product` 商品

| 字段 | 类型 | 允许空 | 默认 | 索引 | 注释 |
|------|------|--------|------|------|------|
| id | BIGINT UNSIGNED | NO | AUTO | PK | 主键 |
| seller_id | BIGINT UNSIGNED | NO | - | IDX(seller, status) | 卖家 |
| category_id | BIGINT UNSIGNED | NO | - | IDX(category, status) | 分类 |
| title | VARCHAR(64) | NO | - | - | 标题 |
| description | TEXT | YES | '' | - | 描述（≤500 字） |
| price | DECIMAL(10,2) | NO | - | - | 售价 |
| original_price | DECIMAL(10,2) | YES | NULL | - | 原价 |
| condition | VARCHAR(16) | NO | 'like_new' | - | new / like_new / good / fair |
| status | VARCHAR(16) | NO | 'pending' | IDX(status, -created_at) | 状态机 |
| school | VARCHAR(64) | YES | '' | - | 商品所在学校（冗余 seller.school） |
| view_count | INT | NO | 0 | - | 浏览数 |
| favorite_count | INT | NO | 0 | - | 收藏数（冗余） |
| audit_remark | VARCHAR(128) | YES | '' | - | 审核备注 |
| audited_at | DATETIME | YES | NULL | - | 审核时间 |
| created_at | DATETIME | NO | now() | - | 创建时间 |
| updated_at | DATETIME | NO | now() ON UPDATE | - | 更新时间 |
| sold_at | DATETIME | YES | NULL | - | 售出时间 |

**外键**：`seller_id` ON DELETE CASCADE（删除卖家级联删除商品）；`category_id` ON DELETE PROTECT（保护分类不误删）。

**复合索引**：

- `idx_mkt_prod_status_ctime (status, -created_at)` — 首页瀑布流"在售+最新"
- `idx_mkt_prod_cat_status (category_id, status)` — 类目页过滤
- `idx_mkt_prod_seller_status (seller_id, status)` — "我的发布"

**典型查询**：

```sql
-- 首页瀑布流：on_sale + 按时间倒序 + 关键词
SELECT p.id, p.title, p.price, p.cover_url, u.school
FROM   market_product p
JOIN   market_user u ON u.id = p.seller_id
WHERE  p.status = 'on_sale'
  AND  (p.title LIKE '%关键词%' OR p.description LIKE '%关键词%')
ORDER  BY p.created_at DESC
LIMIT  20 OFFSET 0;

-- 相似商品（同分类，排除自己）
SELECT id, title, price FROM market_product
WHERE category_id = ? AND status = 'on_sale' AND id <> ?
ORDER BY created_at DESC LIMIT 6;
```

### 4.5 `market_favorite` 收藏

| 字段 | 类型 | 允许空 | 默认 | 索引 | 注释 |
|------|------|--------|------|------|------|
| id | BIGINT UNSIGNED | NO | AUTO | PK | 主键 |
| user_id | BIGINT UNSIGNED | NO | - | UK(user,product) | 收藏者 |
| product_id | BIGINT UNSIGNED | NO | - | UK(user,product) | 商品 |
| created_at | DATETIME | NO | now() | - | 收藏时间 |

**唯一约束**：`unique_together = (user_id, product_id)`，重复收藏由 DB 兜底。

**外键**：`user_id` CASCADE；`product_id` CASCADE。

**典型查询**：

```sql
-- 我的收藏列表
SELECT p.* FROM market_favorite f
JOIN   market_product p ON p.id = f.product_id
WHERE  f.user_id = ?
ORDER  BY f.created_at DESC;
```

| seller_id | BIGINT UNSIGNED | NO | - | IDX(seller, status) | 卖家 |
| status | VARCHAR(16) | NO | 'requested' | - | 状态机 |
| shipping_method | VARCHAR(16) | NO | 'pickup' | - | pickup / express |
| price | DECIMAL(10,2) | NO | - | - | 成交价快照 |

**外键**：`product_id` SET_NULL（商品被删后保留订单记录，仅清空 product_id）；`buyer_id` / `seller_id` PROTECT（防止误删有订单的用户）。

**复合索引**：

- `idx_mkt_ord_buyer_status (buyer_id, status)` — 我的购买
- `idx_mkt_ord_seller_status (seller_id, status)` — 我的销售

**价格快照**：`price` 在创建订单时拷贝自 `product.price`，避免商品后续改价影响订单。

## 5. 索引设计

### 5.1 索引清单

| 索引名 | 表 | 字段 | 类型 | 覆盖场景 |
|--------|----|------|------|----------|
| PRIMARY | 全部 | id | 主键 | - |
| uk_mkt_user_username | market_user | username | UNIQUE | 登录查询 |
| idx_mkt_user_role_active | market_user | role, is_active | 复合 | 管理员列表 |
| uk_mkt_category_code | market_category | code | UNIQUE | 按 code 引用 |
| idx_mkt_cat_parent_sort | market_category | parent_id, sort_order | 复合 | 类目树 |
| idx_mkt_prod_status_ctime | market_product | status, -created_at | 复合 | 首页瀑布流 |
| idx_mkt_prod_cat_status | market_product | category_id, status | 复合 | 类目页 |
| idx_mkt_prod_seller_status | market_product | seller_id, status | 复合 | 我的发布 |
| uk_mkt_fav_user_product | market_favorite | user_id, product_id | UNIQUE | 防重复 |
| uk_mkt_conv_product_buyer | market_conversation | product_id, buyer_id | UNIQUE | 防重复会话 |
| idx_mkt_msg_conv_time | market_message | conversation_id, created_at | 复合 | 聊天时间线 |

| idx_mkt_rp_status_ctime | market_report | status, -created_at | 复合 | 举报处理列表 |
| idx_mkt_aud_operator | market_audit_log | operator_id, -created_at | 复合 | 操作历史 |
| uk_mkt_setting_key | market_system_setting | key | UNIQUE | 配置查询 |

### 5.2 索引选型原则

- **等值 + 范围**：将等值字段放复合索引左侧，范围字段放右侧。例如 `(status, -created_at)` 覆盖 `WHERE status=? ORDER BY created_at DESC`。
- **排序与 LIMIT**：把排序键纳入索引，可避免 filesort。
- **覆盖索引**：v1 不引入（数据量小，命中率足够）。

## 6. 状态机字段说明

### 6.1 Product 状态机

```
draft  --提交审核-->  pending
pending --审核通过--> on_sale
pending --审核驳回--> draft  (audit_remark 写入驳回理由)
on_sale --有人下单--> pending_sold
pending_sold --订单取消--> on_sale
pending_sold --订单完成--> sold
on_sale --卖家下架--> off_shelf
off_shelf --重新上架--> on_sale
sold --（终态）--> [*]
```

详细迁移函数见 [CM-LLD-001 §3.1](#)。

### 6.2 Order 状态机

```
requested  --卖家确认-->  confirmed
confirmed  --开始自取/发货--> shipping
shipping   --确认收货-->  completed
任意阶段    --取消-->        cancelled
completed  --（终态，触发评价）--> [*]
cancelled  --（终态）--> [*]
```

### 6.3 Report 状态机

```
pending  --warn-->    warned
pending  --remove-->  removed  (商品下架)
pending  --ban-->     banned   (卖家封禁)
pending  --reject-->  rejected
*        --（终态）--> [*]
```

### 6.4 状态机不变量

- `Product.status='sold'` 时，`sold_at` 必填。
- `Order.status='completed'` 时，`completed_at` 必填。
- `Order.status='cancelled'` 时，写入 `AuditLog(action='reject_order' OR 'cancel_order')`。
- `Report.status` 一旦离开 `pending`，`handled_at` + `handler_id` 必填。

## 9. 数据迁移与回滚策略

### 9.1 迁移工具

使用 Django 原生 `migrate`：

```bash
python manage.py makemigrations market
python manage.py migrate market
```

### 9.2 迁移命名

`<n>_<verb>_<target>.py`，例如 `0003_report_action_remark.py`。

### 9.3 回滚

```bash
# 回滚 1 个 migration
python manage.py migrate market 0002
```

### 9.4 大表变更

- 添加列：先加可空列 → 写脚本回填 → NOT NULL（拆 3 个 migration）
- 添加索引：`AddIndexOperation` 走 MySQL `ALTER TABLE`；高峰期（08:00-22:00）禁止 DDL
- 删列：先代码兼容 → 灰度删 → 监控一周 → 真正删

## 12. 安全与合规

### 12.1 密码

- Django `pbkdf2_sha256` 哈希（默认 600,000 轮）
- 不在日志中输出明文密码
- 登录失败限速：5 次 / 15 分钟 / IP（[CM-LLD-001 §5](#)）

### 12.2 个人信息

- 学号、邮箱仅本人和管理员可见
- 软删除用户不暴露（`is_active=False` 后不能登录）
- 不在审计日志中记录密码字段

### 12.3 SQL 注入

- 全部使用 ORM 参数化查询
- 禁止裸 SQL 字符串拼接

### 12.4 XSS / CSRF

- Web 后台启用 CSRF（Django 默认）
- 小程序使用 JWT 不涉及 CSRF
- 富文本字段（description）v1 走白名单转义，不允许 `<script>`

### 12.5 数据脱敏

- 审计日志中手机号 / 邮箱打码：`138****1234`
- 演示数据使用 `demo_001 ~ demo_010`
