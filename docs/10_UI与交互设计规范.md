## 1. 设计原则与品牌定位

### 1.1 品牌定位

| 维度 | 描述 |
|------|------|
| **品牌名** | 易物（暂定） |
| **目标用户** | 18-26 岁高校学生 / 研究生 |
| **品牌调性** | 活力、年轻、可信、真实 |
| **核心色彩** | 品牌橙 `#FF6B35` |
| **辅助色** | 微信绿 `#07C160`（成功）、警示黄 `#FFA500`、错误红 `#FF4D4F` |
| **Logo 风格** | 极简线条 + Lucide 图标风格 |

### 1.2 设计原则

| 原则 | 描述 | 落地动作 |
|------|------|---------|
| **内容优先** | 商品图 > 价格 > 标题 > 描述 | 商品卡片主图占 70% 高度；价格字号 17px bold 高于标题 14px |
| **强 CTA** | 交易按钮使用品牌橙，尺寸 ≥ 44pt | 主按钮 `#FF6B35` + 高度 88rpx / 48px |
| **微动效** | 200ms ease-out 是基线节奏 | 全局 transition token `--duration-base: 200ms` |
| **二手感** | 弱化装饰，强调真实感 | 必显成色标签、信用分、距离、学校等"实拍向"信息 |
| **去冗余** | 同色不重复、间距有节奏 | 全局卡片间距 12px、页面边距 16px |
| **可访问** | 4.5:1 对比度、44pt 触控、focus 环 | 主文本与背景对比度 ≥ 7:1 |

### 1.3 三大场景视觉差异化

| 场景 | 主色 | 风格 | 强调信息 |
|------|------|------|----------|
| **浏览/发现** | 中性灰 + 品牌橙点缀 | 卡片瀑布流、宽松节奏 | 商品图、价格、距离 |
| **交易/撮合** | 品牌橙 + 微信绿 | 大按钮、明确指引 | CTA、订单状态、倒计时 |
| **管理/审核** | 中性灰 + 状态色 | 表格、信息密度高 | 数据、状态、操作 |

### 1.4 竞品参考

| 竞品 | 取其精华 |
|------|----------|
| 闲鱼 | 扁平化、强品牌色、商品卡瀑布流 |
| 得物 | 黑白为主、单点强调色、1du=4pt 单位制 |
| 小红书 | 双列瀑布流、错落节奏、社区氛围 |
| 微信小程序原生 | 国民认知度高的状态色 |

## 2. 设计 Token 引用规范

> 本节是「使用约定」。所有 token 实际值参见 [2026-06-06-design-tokens.md](file:///d:/文件/工作%20作业/微信小程序实训/4次课程内容/综合实训/docs/superpowers/specs/2026-06-06-design-tokens.md)。

### 2.1 三层结构

```
┌──────────────────────────────────────────────────────┐
│  Primitive Layer（原子层）                             │
│  - color.brand.500   = #FF6B35                       │
│  - color.gray.900    = #1A1A1A                       │
│  - space.4           = 16px / 32rpx                  │
│  - radius.md         = 12px                          │
└──────────────────────┬───────────────────────────────┘
                       │  语义化
┌──────────────────────▼───────────────────────────────┐
│  Semantic Layer（语义层）                              │
│  - color.action.primary  -> color.brand.500          │
│  - color.text.body       -> color.gray.900           │
│  - color.bg.surface      -> color.white              │
└──────────────────────┬───────────────────────────────┘
                       │  组件化
┌──────────────────────▼───────────────────────────────┐
│  Component Layer（组件层）                             │
│  - Button.bg.primary     -> color.action.primary     │
│  - Card.bg               -> color.bg.surface         │
│  - Tag.bg.ai             -> color.brand.50           │
└──────────────────────────────────────────────────────┘
```

### 2.2 引用规范

#### 2.2.1 Web 端（Vue 3 + Vite）

```vue
<style scoped>
.btn-primary {
  background: var(--color-primary);
  color: var(--color-text-inverse);
  height: 48px;
  border-radius: var(--radius-md);
  padding: 0 var(--space-4);
  font-size: var(--font-size-md);
  font-weight: var(--font-weight-semibold);
  transition: background var(--duration-fast) var(--ease-out);
}
</style>
```

#### 2.2.2 小程序端（WXSS）

```css
/* miniprogram/app.wxss */
.btn-primary {
  background: var(--color-primary);
  color: var(--color-text-inverse);
  height: 88rpx;
  border-radius: var(--radius-md);
  padding: 0 var(--space-4);
  font-size: var(--font-size-md);
  font-weight: var(--font-weight-semibold);
  transition: background var(--duration-fast) var(--ease-out);
}
```

#### 2.2.3 禁止事项

- ❌ 硬编码颜色：`color: #FF6B35`（必须用 `var(--color-primary)`）。
- ❌ 硬编码尺寸：`width: 16px`（必须用 `var(--space-2)`）。
- ❌ 跨端混用单位：小程序中用 px 而非 rpx。
- ❌ 局部覆盖 token：组件中**不得**定义新的 `--xxx` 变量。

### 2.3 Token 变更流程

```
设计师在 Figma 修改 token
        ↓
PR 提交 docs/superpowers/specs/2026-06-06-design-tokens.md
        ↓
评审通过后同步到：
  - frontend-web/src/style.css
  - frontend-admin/src/style.css
  - miniprogram/app.wxss
        ↓
影响面评估：扫描所有组件引用，更新受影响页面
        ↓
走完整端到端回归测试
```

## 3. 色彩系统

### 3.1 主品牌色

| Token | Hex | 用途 |
|-------|-----|------|
| `--color-primary` | `#FF6B35` | 主 CTA、强调、价格 |
| `--color-primary-hover` | `#E55A2B` | 按钮 hover/press |
| `--color-primary-active` | `#CC4A1F` | 按下态 |
| `--color-primary-soft` | `#FFE5DA` | 选中背景、徽章底、标签底 |
| `--color-primary-disabled` | `#FFB499` | 禁用态 |

**使用决策**：

| 场景 | 推荐 | 不推荐 |
|------|------|--------|
| 主 CTA「我想要」「私聊议价」 | `--color-primary` | 灰色（按钮低优先） |
| 商品价格 | `--color-text-price` (`#FF4D4F`) | `--color-primary`（避免与按钮撞色） |
| 选中态（标签 / Tab） | `--color-primary-soft` 底 + `--color-primary` 文 | 实色底 |
| 禁用态 | `--color-primary-disabled` | 灰底 |

### 3.2 文本色

| Token | Hex | 对比度 | 用途 |
|-------|-----|--------|------|
| `--color-text-primary` | `#1A1A1A` | 16.1:1 | 标题、详情正文 |
| `--color-text-secondary` | `#666666` | 5.7:1 | 描述、元信息 |
| `--color-text-tertiary` | `#999999` | 2.85:1 | 辅助（仅 ≥ 14px） |
| `--color-text-disabled` | `#CCCCCC` | 1.6:1 | 禁用（不得承载重要信息） |
| `--color-text-inverse` | `#FFFFFF` | - | 深色背景上的文字 |
| `--color-text-link` | `#FF6B35` | 4.6:1 | 链接 |
| `--color-text-price` | `#FF4D4F` | 4.7:1 | 价格专用红 |

### 3.3 背景色

| Token | Hex / RGBA | 用途 |
|-------|-----------|------|
| `--color-bg-page` | `#F5F5F7` | 页面底色 |
| `--color-bg-card` | `#FFFFFF` | 卡片、商品卡 |
| `--color-bg-section` | `#FAFAFA` | 区块底色 |
| `--color-bg-mask` | `rgba(0,0,0,0.4)` | 弹窗遮罩 |
| `--color-bg-hover` | `#F0F0F0` | hover 态 |
| `--color-bg-pressed` | `#E5E5E5` | 按下态 |

### 3.4 状态色

| Token | Hex | 用途 |
|-------|-----|------|
| `--color-success` | `#07C160` | 成功、信用分高、操作完成 |
| `--color-warning` | `#FFA500` | 提示、待处理、擦边词 |
| `--color-error` | `#FF4D4F` | 错误、信用分低、违规 |
| `--color-info` | `#1989FA` | 信息提示、链接辅助 |

### 3.5 信用分等级

| Token | 范围 | Hex | 标签 |
|-------|------|-----|------|
| `--color-credit-high` | ≥ 90 | `#07C160` | 信用极好 |
| `--color-credit-mid` | 60-89 | `#FFA500` | 信用良好 |
| `--color-credit-low` | < 60 | `#FF4D4F` | 信用一般 |

**徽章规则**：
- 形状：圆形（`--radius-full`），尺寸 56×56rpx（小程序）/ 28×28px（Web）
- 内部：分数数字（大）+ "信用" 文案（小）
- 描边：2rpx / 1px 等级色
- 适用：商品详情、卖家卡片、消息列表头像

### 3.6 边框 / 分隔线

| Token | Value | 用途 |
|-------|-------|------|
| `--color-border-light` | `#EEEEEE` | 卡片内分隔、弱边界 |
| `--color-border-base` | `#DDDDDD` | 常规边框、输入框 |
| `--color-border-strong` | `#CCCCCC` | 强调边框 |
| `--color-divider` | `#F0F0F0` | 列表分割线 |

### 3.7 暗色模式（v1.1 规划）

| Token | 暗色值 | 备注 |
|-------|--------|------|
| `--color-bg-page` | `#121212` | 主底 |
| `--color-bg-card` | `#1E1E1E` | 卡片底 |
| `--color-text-primary` | `#F0F0F0` | 主文本反转 |
| `--color-text-secondary` | `#B0B0B0` | 次文本反转 |

| Token | 小程序 (rpx) | Web (px) | 用途 |
|-------|-------------|----------|------|
| `--space-0` | 0rpx | 0px | 复位 |
| `--space-1` | 8rpx | 4px | 紧凑（图标与文字） |
| `--space-2` | 16rpx | 8px | 元素内距、小间距 |
| `--space-3` | 24rpx | 12px | 卡片内距、瀑布流行间距 |
| `--space-4` | 32rpx | 16px | 区块内距、页面左右内边距 |
| `--space-5` | 40rpx | 20px | 章节间距 |
| `--space-6` | 48rpx | 24px | 大区块 |
| `--space-8` | 64rpx | 32px | 页面外边距 |
| `--space-10` | 80rpx | 40px | 节段 |
| `--space-12` | 96rpx | 48px | 大节段 |
| `--space-16` | 128rpx | 64px | 启动页留白 |

### 5.2 关键场景间距

| 场景 | Token |
|------|-------|
| 页面左右内边距 | `--space-4` |
| 卡片间距（瀑布流 row-gap） | `--space-3` |

| 区块间距 | `--space-6` |
| 按钮之间间距 | ≥ `--space-2` |
| Tab 间距 | `--space-3` |

### 5.3 圆角体系

| Token | Value | 用途 |
|-------|-------|------|
| `--radius-sm` | 4px | 标签、徽章、checkbox |
| `--radius-base` | 8px | 输入框、小按钮 |
| `--radius-md` | 12px | 卡片、商品卡、按钮 |
| `--radius-lg` | 16px | 弹窗、底单、抽屉 |
| `--radius-xl` | 24px | 大弹窗 |
| `--radius-2xl` | 32px | 特殊大卡片 |
| `--radius-full` | 9999px | 头像、胶囊按钮、信用分徽章 |

### 5.4 圆角使用决策

- 商品卡片：`--radius-md`（12px）
- 主按钮：`--radius-md`（12px） 或 `--radius-full`（胶囊型，Tab 切换时使用）
- 输入框：`--radius-base`（8px）
- 标签 / 徽章：`--radius-sm`（4px）
- 头像：`--radius-full`（9999px）

## 6. 阴影与动效

### 6.1 阴影体系

| Token | Value | 用途 |
|-------|-------|------|
| `--shadow-none` | none | 复位 |
| `--shadow-sm` | `0 1rpx 4rpx rgba(0,0,0,0.04)` | 列表项、内嵌 |
| `--shadow-base` | `0 2rpx 12rpx rgba(0,0,0,0.06)` | 卡片（默认） |
| `--shadow-md` | `0 4rpx 24rpx rgba(0,0,0,0.08)` | 弹窗、Toast |
| `--shadow-lg` | `0 8rpx 32rpx rgba(0,0,0,0.12)` | 抽屉、Modal |
| `--shadow-orange` | `0 4rpx 16rpx rgba(255,107,53,0.3)` | 主按钮（带品牌色光晕） |
| `--shadow-inset` | `inset 0 1rpx 2rpx rgba(0,0,0,0.05)` | 内嵌、输入框聚焦态 |

### 6.2 动效体系

| Token | Value | 用途 |
|-------|-------|------|
| `--duration-fast` | 150ms | 微交互、状态切换、颜色变化 |
| `--duration-base` | 200ms | 元素进入、卡片浮现、按钮反馈 |
| `--duration-slow` | 300ms | 弹窗、抽屉、Modal |
| `--duration-slower` | 500ms | 页面切换 |
| `--ease-out` | `cubic-bezier(0.2, 0, 0, 1)` | 进入（推荐默认） |
| `--ease-in` | `cubic-bezier(0.4, 0, 1, 1)` | 退出 |
| `--ease-in-out` | `cubic-bezier(0.4, 0, 0.2, 1)` | 双向往返 |
| `--ease-spring` | `spring(0.6, 80, 10)` | 弹性（仅小程序） |

### 6.3 动效规范

1. 入场用 `ease-out`，退场用 `ease-in`，**禁止反向**。
2. 动效超过 300ms **必须**支持 `prefers-reduced-motion`。
3. 不使用 `linear`（除进度条外）。
4. 按钮按下：背景色 + 缩放 `scale(0.98)` + 100ms。
5. 卡片 hover：阴影从 `--shadow-base` 升级到 `--shadow-md` + 200ms。

## 7. 图标系统

### 7.1 来源

- **图标库**：Lucide（https://lucide.dev/icons/）
- **设计规范**：`stroke-width: 2px`、`size: 24px`、outline 风格
- **Web 端包**：`lucide-vue-next`（Vue 3）或 `@lucide/react`
- **小程序端**：使用 base64 内联 SVG 或 iconfont 自建

### 7.2 尺寸

| Token | Value | 用途 |
|-------|-------|------|
| `--icon-xs` | 12px | 内嵌文字旁 |
| `--icon-sm` | 16px | 标签内 |
| `--icon-base` | 20px | 按钮内、列表项 |
| `--icon-md` | 24px | 导航、卡片标题 |
| `--icon-lg` | 32px | 大型 CTA、引导 |
| `--icon-xl` | 48px | 空状态、启动页 |

### 7.3 风格

- **默认**：outline（线性）
- **强调态**：filled（实心，仅用于主 CTA 图标、收藏激活态）
- **描边宽度**：统一 1.5px（自定义时）或 2px（Lucide 默认）
- **颜色**：默认 `--color-text-primary`，激活 `--color-primary`

### 7.4 禁止事项

> **绝对禁止**：
> 1. 使用 emoji 作为 UI 图标（用户规则 5）。
> 2. 混用不同描边宽度的图标。
> 3. 在 16px 尺寸下使用实心图标（视觉过重）。
> 4. 使用彩色 emoji 表情（功能区域）—— 仅聊天内容允许用户输入 emoji 文字。

## 15. 状态机与微交互

### 15.1 商品状态机（与后端一致）

```
draft → pending → on_sale → sold
                    ↓
                 off_shelf
```

**UI 状态映射**：

| 状态 | 列表展示 | 详情页 | 操作按钮 |
|------|----------|--------|----------|
| `draft` | 隐藏 | 不可见 | 「编辑」「提交审核」 |
| `pending` | 「审核中」灰色标签 | 不可见 | 「撤回」「查看进度」 |
| `on_sale` | 正常卡片 | 正常详情 | 「私聊议价」「立即购买」 |
| `off_shelf` | 「已下架」置灰 | 可见 | 「重新上架」 |
| `sold` | 「已售出」红色标签 | 可见 | 「查看订单」「类似推荐」 |

### 15.2 订单状态机

```
requested → confirmed → shipped → completed
        ↘  rejected ↗
        ↘  cancelled
```

**UI 状态映射**：

| 状态 | 买家按钮 | 卖家按钮 |
|------|----------|----------|
| `requested` | 「取消申请」 | 「确认」「拒绝」 |
| `confirmed` | 「提醒发货」 | 「发货」 |
| `shipped` | 「确认收货」 | 「查看物流」 |
| `completed` | 「评价」「再次购买」 | 「评价买家」 |
| `rejected` | 「查看原因」 | 「查看原因」 |
| `cancelled` | 「重新申请」 | 无 |

### 15.3 按钮微交互

- 默认态：背景 `--color-primary`，文字 `--color-text-inverse`。
- Hover：背景 `--color-primary-hover`，过渡 150ms。
- Press：背景 `--color-primary-active`，缩放 `scale(0.98)`，过渡 100ms。
- Disabled：背景 `--color-primary-disabled`，文字 `--color-text-disabled`。
- Loading：原位置替换为 spinner 旋转。

### 15.4 卡片微交互

- 列表滚动：卡片入场透明度 0 → 1，时长 200ms，错开 30ms。
- Hover：阴影从 `--shadow-base` 升级到 `--shadow-md`，过渡 200ms。
- 点击：阴影从 `--shadow-md` 升级到 `--shadow-sm`，过渡 100ms。

### 15.5 表单微交互

- 输入框聚焦：边框 `--color-primary`，1px → 2px 过渡 150ms；外圈 `--shadow-inset`。
- 输入校验失败：边框 `--color-error`，抖动动画 200ms。
- 提交按钮：loading 时显示 spinner，文字替换为「提交中...」。

### 15.6 AI 抽屉微交互

- 抽屉进入：从下往上滑入，时长 300ms，`ease-out`。
- 抽屉退出：滑出到底部，时长 200ms，`ease-in`。
- AI loading：抽屉内显示 3 个跳动的小圆点（dot-pulse），循环 1.2s。
- AI 降级：抽屉顶部加灰色横条（高度 32rpx）滑入。

## 16. 无障碍与国际化

### 16.1 无障碍要求

- 对比度：主文本 ≥ 7:1，次文本 ≥ 4.5:1，禁用文字不得承载重要信息。
- 触控热区：≥ 44×44pt。
- 焦点环：所有可聚焦元素必须有 `outline: 2px solid var(--color-primary)`。
- 替代文本：所有 `<img>` 必须有 `alt` 属性；图标按钮必须有 `aria-label`。
- 语义化标签：使用 `<header>` / `<nav>` / `<main>` / `<footer>` 等语义化标签。
- 跳过导航：提供「跳到主内容」链接。

### 16.2 国际化（v1.1）

- v1.0 仅中文。
- v1.1 引入 i18n 框架（vue-i18n / Taro i18n）。
- 货币、日期、数字格式本地化。
- LTR / RTL 支持（阿拉伯语等）。

### 16.3 动效降级

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 17. 性能与质量基线

### 17.1 性能指标

| 指标 | 目标 |
|------|------|
| 首屏渲染 | ≤ 1.5s（小程序）/ ≤ 2s（Web） |
| LCP（Largest Contentful Paint） | ≤ 2.5s |
| FID（First Input Delay） | ≤ 100ms |
| CLS（Cumulative Layout Shift） | ≤ 0.1 |
| 包体积 | 小程序 ≤ 4MB；Web 首屏 ≤ 500KB |
| 图片懒加载 | 列表卡片默认懒加载 |

### 17.2 质量基线

- TypeScript 严格模式（`strict: true`）。
- ESLint + Prettier 强制。
- 单元测试覆盖率 ≥ 70%（核心组件 ≥ 90%）。
- E2E 测试：核心流程 100% 覆盖（发布、购买、私聊）。
- 无障碍审查：每季度一次自动化扫描 + 人工检查。

### 17.3 性能优化手段

- 图片：WebP 优先 + CDN 缩略图 + 懒加载。
- 列表：虚拟滚动（`@tanstack/vue-virtual` / 小程序 `recycle-view`）。
- 路由：懒加载 + 预取下一页。
- 状态：Pinia 持久化 + computed 缓存。
- 打包：Terser + esbuild + split chunks。

## 19. 修订与变更流程

### 19.1 Token 变更流程

```
设计师 / 前端 PR
  ↓
更新 docs/superpowers/specs/2026-06-06-design-tokens.md
  ↓
同步到 3 个代码库：
  - frontend-web/src/style.css
  - frontend-admin/src/style.css
  - miniprogram/app.wxss
  ↓
影响面扫描：grep token 引用
  ↓
更新相关组件
  ↓
UI 评审（设计师 + 前端 Lead）
  ↓
合并 + 跑视觉回归（VRT）
```

### 19.2 组件变更流程

```
需求 / 缺陷
  ↓
UI 设计师出 Figma 设计稿
  ↓
前端实现（参考本规范）
  ↓
自测 + Code Review
  ↓
设计走查（截图对比 Figma）
  ↓
合并
```

### 19.3 兼容性策略

- **新增 Token**：minor 版本，不影响老页面。
- **重命名 Token**：major 版本，提供 codemod 工具。
- **删除 Token**：先 deprecate 1 个版本，再删除。
- **修改 Token 值**：仅当影响范围评估通过后修改。
