## 一、启动顺序

### 1.1 基础设施确认

- [ ] MySQL 已启动，库 `campus_market` 已建（执行 `backend\scripts\create_mysql_db.sql`）
- [ ] RabbitMQ 已启动（可选，路径 `D:\RJ\RabbitMQ`），用于订单异步通知
- [ ] `backend\.env` 已配置：`DB_*` / `LLM_API_KEY`（可选） / `MEDIA_ROOT` / `RABBITMQ_URL`（可选）

### 1.2 启动三端

1. **后端**：`scripts\start-backend.bat`（端口 8000，5 秒内启动完成）
2. **Web 卖家工作台**：`scripts\start-frontend-web.bat`（端口 3000）
3. **Web 管理后台**：`scripts\start-frontend-admin.bat`（端口 5173）
4. **微信小程序**：微信开发者工具 → 导入 `miniprogram\`，修改 `app.js` 的 `apiBase`，勾选「不校验合法域名」

### 1.3 一键启动 / 停止

```bat
scripts\start-all.bat    :: 一键启动全部
scripts\stop-all.bat     :: 一键停止全部
```

## 九、常见联调问题

| 问题 | 解决方案 |
|------|----------|
| `40101` Token 失效 | 重新登录获取 access token |
| `40301` 权限不足 | 切换为对应角色账号 |
| `40401` 资源不存在 | 检查 id 是否正确（注意大小写） |
| CORS 报错 | 后端已默认开启 corsheaders；检查前端 `.env.development` 的 `VITE_API_BASE` |
| 小程序图片不显示 | 后端 `/media/` 路径需在 `ALLOWED_HOSTS` 或小程序 downloadFile 合法域名中 |
| AI 返回 mock | 检查 `LLM_API_KEY` 是否配置；mock 标识为"AI 推荐"灰色 |
| RabbitMQ 消息未消费 | 检查 `D:\RJ\RabbitMQ` 是否启动；查看后端日志 |
