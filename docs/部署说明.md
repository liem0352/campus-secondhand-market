## 方案 A：课堂演示（推荐）

| 组件 | 命令 / 配置 |
|------|-------------|
| Django | `python manage.py runserver 0.0.0.0:8000` |
| Web 卖家工作台 | `npm run dev`（Vite 代理 `/api` → 8000，端口 3000） |
| Web 管理后台 | `npm run dev`（端口 5173） |
| 微信小程序 | `apiBase` 用 `127.0.0.1`（模拟器）或电脑局域网 IP（真机） |
| RabbitMQ（可选） | 本地路径 `D:\RJ\RabbitMQ`，订单异步通知用 |

**一键启动**：

```bat
scripts\install-deps.bat     :: 首次安装依赖
scripts\start-all.bat        :: 启动后端 + 两个 Web 端
scripts\stop-all.bat         :: 停止全部
```

**真机调试**：手机与电脑同一 WiFi，`ipconfig` 查 IP，防火墙放行 8000 端口；微信开发者工具勾选「不校验合法域名」。

## 二、媒体文件（`MEDIA_ROOT`）注意事项

二手交易平台有**大量用户上传图片**（商品图、头像、消息图、举报截图），媒体文件的配置与部署必须谨慎处理。

### 2.1 `backend/config/settings.py` 配置

```python
import os
from pathlib import Path

BASE_DIR = Path(__file__).resolve().parent.parent

MEDIA_URL = '/media/'
MEDIA_ROOT = BASE_DIR / 'media'

# 上传限制（5MB）
DATA_UPLOAD_MAX_MEMORY_SIZE = 5 * 1024 * 1024
FILE_UPLOAD_MAX_MEMORY_SIZE = 5 * 1024 * 1024

# 允许的图片格式
ALLOWED_IMAGE_TYPES = ['jpg', 'jpeg', 'png', 'webp']
```

### 2.2 上传目录结构

```
backend/media/
├── avatars/           # 用户头像（YYYY/MM/）
│   └── 2026/06/abc123.jpg
├── products/          # 商品图（YYYY/MM/）
│   └── 2026/06/def456-0.jpg
│   └── 2026/06/def456-1.jpg
├── messages/          # 私聊图片
└── reports/           # 举报截图
```

### 2.3 生产环境部署建议

| 场景 | 建议 |
|------|------|
| **单机部署** | `MEDIA_ROOT` 指向本地目录，Nginx 直接 `alias` |
| **多机部署 / 集群** | 使用**共享存储**（NFS / OSS / MinIO），所有 Django 实例挂载到同一路径 |
| **云原生** | 直接对接对象存储（阿里云 OSS / AWS S3 / 腾讯云 COS），通过 `django-storages` 重写 `Storage` |
| **CDN 加速** | 阿里云 OSS / 腾讯云 COS 绑定 CDN，回源到对象存储 |

### 2.4 Nginx `location /media/` 示例

```nginx
location /media/ {
    alias D:/综合实训/backend/media/;
    expires 7d;              # 浏览器缓存 7 天
    add_header Cache-Control "public, immutable";
}
```

> ⚠️ **注意**：开发环境下 Django `runserver` 自动处理 `/media/`，生产环境**必须**由 Nginx / Apache 托管。

## 三、AI 配置（`.env` 中的 LLM_API_KEY）

校园二手交易平台的 AI 能力（AI 一键发布 / 议价参考 / 内容审核）依赖 LLM。配置在 `backend/.env`：

```ini
# === LLM 配置（OpenAI 兼容协议）===
LLM_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o-mini           # 支持多模态（图片输入）
LLM_TIMEOUT=30                   # 秒
LLM_TEMPERATURE=0.4              # 略低于 0.5 让输出更稳定
LLM_MOCK_MODE=False              # True 强制 mock（演示用）

# === AI 服务开关 ===
AI_PUBLISH_ASSIST_ENABLED=True   # AI 一键发布
AI_PRICE_SUGGEST_ENABLED=True    # 议价参考价
AI_MODERATE_ENABLED=True         # 内容审核
```

### 3.1 降级策略

- 未配置 `LLM_API_KEY` → 自动 mock
- `LLM_MOCK_MODE=True` → 强制 mock（用于课堂演示）
- LLM 调用超时 / 失败 → 返回 mock 数据 + 日志告警
- 前端展示"AI 推荐"灰色标识区分真伪

### 3.2 Prompt 模板

集中在 `backend/market/services/ai_prompts.py`，便于调优。详见 `market/services/ai_prompts.py` 的注释。

## 五、RabbitMQ 集成（订单异步通知）

校园二手交易平台的订单状态变更、AI 任务、消息推送都依赖 RabbitMQ 做异步解耦。

### 5.1 本地 RabbitMQ

- 安装路径：`D:\RJ\RabbitMQ`
- 管理界面：`http://127.0.0.1:15672/`（默认 `guest / guest`）
- AMQP 端口：`5672`

### 5.2 `backend/.env` 配置

```ini
# === RabbitMQ 配置 ===
RABBITMQ_URL=amqp://guest:guest@127.0.0.1:5672//
RABBITMQ_ENABLED=True

# === 队列与交换机 ===
RABBITMQ_EXCHANGE_ORDER=market.order          # 订单事件交换机
RABBITMQ_QUEUE_NOTIFY=market.notify            # 通知队列
RABBITMQ_QUEUE_AI=market.ai                     # AI 任务队列
```

### 5.3 应用场景

| 事件 | 交换机 / 路由键 | 消费者 |
|------|-----------------|--------|
| 订单创建 (`requested`) | `market.order / order.requested` | 通知卖家、推送订阅消息 |
| 订单确认 (`confirmed`) | `market.order / order.confirmed` | 通知买家 |
| 订单完成 (`completed`) | `market.order / order.completed` | 通知双方 + 触发评价提醒 |
| 订单取消 (`cancelled`) | `market.order / order.cancelled` | 通知双方 + 商品回滚到 `on_sale` |
| AI 一键发布 | `market.ai / ai.publish` | Worker 异步调用 LLM（防止阻塞） |
| 内容审核 | `market.ai / ai.moderate` | Worker 异步审核敏感词 |

### 5.4 后端代码位置

- `backend/market/services/mq_publisher.py` — 发布消息
- `backend/market/services/mq_consumer.py` — 消费消息（可独立进程 `python manage.py mq_worker`）

### 5.5 启动 Worker（可选）

```bat
cd backend
C:\Users\liem\AppData\Local\Programs\Python\Python313\python.exe manage.py mq_worker
```

### 5.6 生产部署建议

- **RabbitMQ 集群**：3 节点镜像队列，防止单点故障
- **死信队列**：处理消费失败的消息，避免堆积
- **监控**：通过 RabbitMQ Management API 暴露指标到 Prometheus

## 六、WSGI 服务器

### 6.1 Windows — Waitress

Gunicorn 不支持 Windows，使用 Waitress：

```bat
pip install waitress
cd backend
waitress-serve --port=8000 --threads=4 config.wsgi:application
```

### 6.2 Linux — Gunicorn

```bash
pip install gunicorn
cd /opt/campus_market
source venv/bin/activate
gunicorn config.wsgi:application \
    --bind 0.0.0.0:8000 \
    --workers 4 \
    --timeout 120 \
    --access-logfile logs/access.log \
    --error-logfile logs/error.log
```

## 七、Nginx 配置

### 7.1 安装 Nginx

- **Windows**：下载 http://nginx.org/en/download.html，解压到 `C:\nginx`
- **Linux**：`sudo apt install nginx` 或 `sudo yum install nginx`

### 7.2 完整配置（`nginx.conf`）

```nginx
server {
    listen 80;
    server_name localhost;             # 生产改为正式域名

    # 字符集
    charset utf-8;

    # 卖家工作台静态文件（默认入口）
    location / {
        root D:/综合实训/frontend-web/dist;
        index index.html;
        try_files $uri $uri/ /index.html;   # Vue Router history 模式
    }

    # Django API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        client_max_body_size 10M;            # 图片上传
    }

    # Django Admin
    location /admin/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 媒体文件（商品图 / 头像 / 消息图）
    location /media/ {
        alias D:/综合实训/backend/media/;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # Django 静态文件（collectstatic 收集的 admin 静态资源）
    location /static/ {
        alias D:/综合实训/backend/staticfiles/;
    }
}
```

> 平台管理后台（5173）通常**仅内网访问**，可不挂到 Nginx；如需外网，配置独立的 `server { listen 8080; }` 块。

### 7.3 收集 Django 静态文件

```bat
cd backend
python manage.py collectstatic --noinput
```

### 7.4 启动 Nginx

```bat
cd C:\nginx
start nginx
```

浏览器访问 `http://localhost` → 应该看到卖家工作台登录页。

## 八、Django 生产环境配置检查清单

打开 `backend/config/settings.py`，确认以下生产配置：

```python
# ====== 生产环境安全配置 ======
DEBUG = False                          # ← 必须关闭！
ALLOWED_HOSTS = ['localhost', '127.0.0.1', '你的域名']

# 密钥（生产环境从环境变量读取，绝不能硬编码）
SECRET_KEY = os.environ.get('DJANGO_SECRET_KEY', '不安全默认值')

# 数据库密码（从环境变量读取）
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'campus_market',
        'USER': 'root',
        'PASSWORD': os.environ.get('DB_PASSWORD', 'tyb1124'),
        'HOST': '127.0.0.1',
        'PORT': '3306',
    }
}

# 静态文件
STATIC_ROOT = BASE_DIR / 'staticfiles'
STATIC_URL = '/static/'

# 媒体文件（二手交易平台用户上传的商品图、头像、消息图）
MEDIA_ROOT = BASE_DIR / 'media'
MEDIA_URL = '/media/'

# CORS
CORS_ALLOW_ALL_ORIGINS = False
CORS_ALLOWED_ORIGINS = [
    'https://admin.your-domain.com',
    'https://seller.your-domain.com',
]
```

> **安全警告**：`DEBUG=True` 会暴露源代码和堆栈信息，生产环境必须设为 `False`。数据库密码和 `SECRET_KEY` 必须通过环境变量注入。

## 九、HTTPS 配置（生产建议）

```nginx
server {
    listen 443 ssl http2;
    server_name api.your-domain.com;

    ssl_certificate     /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        # ... 同 7.2
    }
}

# HTTP 强制跳转 HTTPS
server {
    listen 80;
    server_name api.your-domain.com;
    return 301 https://$host$request_uri;
}
```

**微信小程序公众平台** 必须配置 **HTTPS 合法域名**（`api.your-domain.com`）。

## 十二、常见部署问题

| 问题 | 解决方案 |
|------|----------|
| 502 Bad Gateway | Waitress / Gunicorn 未启动，或端口被占用 |
| 静态文件 404 | `python manage.py collectstatic` 未执行 |
| 媒体文件 404 | 检查 Nginx `location /media/` 的 `alias` 路径；Linux 注意 `alias` 末尾 `/` |
| 数据库 1045 错误 | `.env` 中 `DB_PASSWORD` 不正确，或 MySQL 用户权限不足 |
| 数据库 1049 错误 | 库 `campus_market` 未建，执行建库 SQL |
| 上传文件过大 | `client_max_body_size` 太小，设为 `10M` |
| 微信小程序无法访问 | 配置 HTTPS 合法域名，开发期勾选「不校验合法域名」 |
| RabbitMQ 连接失败 | 检查 `RABBITMQ_URL`；Windows 防火墙放行 5672 端口 |
| AI 调用超时 | 调大 `LLM_TIMEOUT`；或暂时开启 `LLM_MOCK_MODE=True` |
