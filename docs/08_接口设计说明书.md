## 1. 概述

### 1.1 目的

本文档定义校园二手交易平台 **RESTful HTTP API** 的完整契约，覆盖微信小程序、Web 卖家工作台、Web 管理后台三端的全部后端交互。

每个业务端点具备：

- HTTP 方法、路径、认证方式、权限
- 查询参数表 / 请求体 JSON Schema
- 成功响应 Schema
- 业务错误码
- curl / fetch 示例

### 1.2 技术栈

| 维度 | 选择 |
|------|------|
| 协议 | HTTPS（开发期可 HTTP） |
| 数据格式 | JSON（`Content-Type: application/json`） |
| 上传 | `multipart/form-data` |
| 鉴权 | JWT（`Authorization: Bearer <access>`） |
| 字符集 | UTF-8 |
| 时间格式 | ISO 8601（`2026-06-15T10:00:00Z`） |
| 金额格式 | 字符串 `"12.00"`（避免 JS 浮点精度问题） |

### 1.3 部署拓扑

```
开发：
  客户端 -> http://127.0.0.1:8000/api
生产：
  客户端 -> https://api.xxx.com/api -> Nginx -> Waitress:8000
```

## 2. 全局约定

### 2.1 统一响应格式

```json
{
  "code": 0,
  "message": "ok",
  "data": { /* 业务数据 */ }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 业务状态码：0=成功，非 0=业务错误 |
| message | string | 文案，可直接给用户展示 |
| data | object/array/null | 业务负载 |

### 2.2 分页

请求：

```
?page=1&page_size=20
```

响应（data 内）：

```json
{
  "total": 487,
  "page": 1,
  "page_size": 20,
  "total_pages": 25,
  "results": [ ... ]
}
```

| 参数 | 默认 | 上限 |
|------|------|------|
| page | 1 | - |
| page_size | 20 | 100 |

### 2.3 排序

```
?ordering=-created_at   # 倒序
?ordering=price         # 升序
?ordering=-view_count,-created_at   # 多键
```

### 2.4 时间格式

- 入参：ISO 8601（`2026-06-15T10:00:00Z`）
- 出参：ISO 8601（后端统一 UTC）
- 客户端展示：本地时区（Asia/Shanghai）

### 2.5 金额

- 全部以字符串返回 `"12.00"`，避免 JS 浮点
- 客户端用 `parseFloat` / BigInt / `decimal.js` 处理

### 2.6 错误响应

```json
{
  "code": 40001,
  "message": "缺少必填字段 title",
  "data": { "fields": { "title": "必填" } }
}
```

### 2.7 幂等性

- 写操作（POST/PUT/PATCH/DELETE）默认**非幂等**
- 涉及订单/支付的端点建议客户端生成 `Idempotency-Key` 头（v2 实现）
- 收藏切换通过 `unique_together` DB 约束实现幂等

## 4. 鉴权接口

### 4.1 `POST /auth/register/`

**请求体**：

```json
{
  "username": "demo_001",
  "password": "Demo@123",
  "email":    "demo_001@school.edu.cn",
  "school":   "示例大学"
}
```

**校验**：

| 字段 | 规则 |
|------|------|
| username | 3-32 字符，`[A-Za-z0-9_]` |
| password | ≥ 8 字符，含字母+数字 |
| email | 可选，邮箱格式 |
| school | 可选，≤ 64 字符 |

**响应 200**：

```json
{
  "code": 0,
  "message": "注册成功",
  "data": {
    "user_id": 2,
    "access":  "eyJ...",
    "refresh": "eyJ..."
  }
}
```

**错误码**：

| code | 说明 |
|------|------|
| 40001 | 参数错误 |
| 40901 | 用户名已存在 |

**curl**：

```bash
curl -X POST http://127.0.0.1:8000/api/auth/register/ \
  -H "Content-Type: application/json" \
  -d '{"username":"demo_001","password":"Demo@123","school":"示例大学"}'
```

### 4.2 `POST /auth/login/`

**请求体**：

```json
{ "username": "demo_001", "password": "Demo@123" }
```

**响应**：

```json
{
  "code": 0,
  "data": {
    "user_id": 2,
    "access":  "eyJ...",
    "refresh": "eyJ..."
  }
}
```

**错误码**：

| code | 说明 |
|------|------|
| 40100 | 账号或密码错误 |
| 40301 | 账号被封禁 |
| 42900 | 登录失败次数过多（5 次/15min） |

### 4.3 `POST /auth/refresh/`

**请求体**：

```json
{ "refresh": "eyJ..." }
```

**响应**：

```json
{ "code": 0, "data": { "access": "eyJ..." } }
```

### 4.4 `POST /auth/logout/`

将 refresh token 加入黑名单（v1 仅记录日志）。

|------|------|------|

| 40100 | 未登录 | 401 |

| 40300 | 无权限 | 403 |
| 40301 | 账号被封禁 | 403 |
| 40302 | 未完成校园认证 | 403 |
| 40400 | 资源不存在 | 404 |
| 40401 | 资源已删除 | 404 |
| 40900 | 状态机冲突 | 409 |
| 40901 | 重复操作 | 409 |

## 17. 限流策略

| Scope | 限制 | 端点 |
|-------|------|------|
| 默认（user） | 300/min/user | 全部 `/api/*` |
| 默认（anon） | 60/min/ip | 全部 `/api/*` |
| AI | 60/min/user | `/api/ai/*` |
| 登录 | 5/15min/ip | `/api/auth/login/` |
| 注册 | 10/hour/ip | `/api/auth/register/` |

超出返回 429 + `Retry-After` 头。
